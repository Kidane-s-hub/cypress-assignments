Page Object Model (POM) ‚Äì Summary
*********************************

Page Object Model (POM) is a design pattern used in test automation to make tests cleaner, reusable, and easier to maintain.

‚≠ê What POM Does:

Separates test steps from page element locators

Keeps all page-related logic in a single place

Makes your test code more readable and organized

üß± How POM Works

You create a Page Object file (example: modalPage.js)
This file contains:

locators

reusable functions

actions for the page

Your test file (example: modalTest.cy.js) only uses those functions.

üéØ Why POM Is Useful

Easier to maintain (change locators in ONE file)

Cleaner test code (tests only contain high-level steps)

Allows reusability (same page functions reused across many tests)

Reduces duplicate code

üìå Example:
Page Object:
modalPage.largeModalButton()

Test:
modalPage.largeModalButton().click();


The test does not need to know the locator #showLargeModal ‚Äî it‚Äôs stored inside the POM.

üß† In short:

POM is a neat way to organize your test automation code so your tests stay clean, readable, and easy to update.
